# Contributing to Research Hub

Thank you for your interest in contributing to Research Hub! This document provides guidelines for contributing data stories and code to the platform.

## How to Contribute

### Contributing Data Stories

1. **Research Standards**: All stories must follow rigorous research standards:
   - Use credible, verifiable data sources
   - Cite all sources properly
   - Maintain objectivity and avoid bias
   - Present evidence-based conclusions

2. **Content Structure**: Follow the scientific paper format:
   - **Abstract**: 150-250 word summary of the research
   - **Introduction**: Background, context, and research objectives
   - **Methodology**: Data sources, collection methods, and analytical approaches
   - **Results**: Key findings with supporting data and visualizations
   - **Discussion**: Interpretation, implications, and limitations
   - **Conclusion**: Summary and future research directions
   - **References**: Complete citations for all sources

3. **Data Visualizations**:
   - Use professional visualization tools (Datawrapper, Flourish, etc.)
   - Ensure visualizations are accessible and clearly labeled
   - Include descriptive captions
   - Verify all data accuracy before publishing

4. **Writing Style**:
   - Use clear, precise language
   - Avoid jargon when possible; define technical terms
   - Write for an educated general audience
   - Maintain professional, academic tone
   - Proofread carefully for grammar and spelling

### Contributing Code

1. **Fork the Repository**: Create your own fork of the project

2. **Create a Branch**: Use descriptive branch names
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Code Standards**:
   - Follow existing code style and conventions
   - Use TypeScript for type safety
   - Write clear, self-documenting code
   - Add comments for complex logic
   - Keep components focused and reusable

4. **Testing**:
   - Test your changes thoroughly
   - Ensure the application builds without errors
   - Test on different screen sizes (responsive design)
   - Check accessibility standards

5. **Commit Messages**:
   - Write clear, descriptive commit messages
   - Use present tense ("Add feature" not "Added feature")
   - Reference issues when applicable

6. **Pull Requests**:
   - Provide a clear description of changes
   - Include screenshots for UI changes
   - Link to related issues
   - Be responsive to feedback

## Categories

Research Hub accepts stories in seven main categories:

1. **Conflict & Wars**: Analysis of conflicts, wars, peacekeeping, and security issues
2. **Environment**: Climate change, conservation, pollution, and sustainability
3. **Technology**: Technological advancement, AI, digital transformation, and innovation
4. **Society**: Social dynamics, demographics, culture, education, and health
5. **Governance**: Government systems, public policy, and administration
6. **Economy**: Economic trends, markets, trade, and financial systems
7. **Politics**: Political systems, elections, parties, and policy analysis

## Data Source Guidelines

### Acceptable Sources

- Government statistical agencies
- International organizations (UN, World Bank, WHO, etc.)
- Peer-reviewed academic journals
- Reputable research institutions
- Official datasets with clear methodology

### Unacceptable Sources

- Unverified social media claims
- Websites without clear authorship or methodology
- Sources with obvious political or commercial bias
- Data without verifiable origins

## Review Process

1. **Initial Review**: Stories are reviewed for format compliance and completeness
2. **Content Review**: Verification of data sources and methodology
3. **Editorial Review**: Check for clarity, grammar, and coherence
4. **Technical Review**: Ensure visualizations work correctly
5. **Publication**: Approved stories are published to the platform

## Citation Format

Use standard academic citation formats. Include:
- Author(s)
- Publication year
- Title
- Publication/source name
- URL (for online sources)
- DOI (when available)

Example:
```
Smith, J., & Johnson, A. (2024). Climate Trends in Southeast Asia.
Journal of Environmental Science, 45(2), 123-145.
https://doi.org/10.xxxx/xxxxx
```

## Ethical Guidelines

1. **Transparency**: Clearly disclose any potential conflicts of interest
2. **Accuracy**: Verify all data and claims before publication
3. **Privacy**: Respect individual privacy and data protection laws
4. **Integrity**: Never fabricate or misrepresent data
5. **Attribution**: Always credit original sources and collaborators

## Questions?

If you have questions about contributing, please:
- Open an issue on GitHub
- Review existing stories for examples
- Consult the README for technical guidance

Thank you for contributing to evidence-based research and data storytelling!
